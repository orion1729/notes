

分號（;）
用分號來分隔多個指令，一旦前面的指令執行完畢
會馬上執行後面的指令
此方式指令的前後並沒有相關性

使用：cmd1;cmd2;cmd3;...


——


&&, ||
這兩種連續執行的符號與分號不同是有相關性的
主要概念就是判斷前面一個 cmd 執行成功與否來決定後面的 cmd 是否要繼續執行
而這個判斷的依據就是 $?，即前一個程式傳回給系統的回傳值
（在 Linux 當中，成功執行的程式是回傳 0 的）

- cmd1 && cmd2
  此符號表示 cmd1 成功執行（$? = 0）後，cmd2 才會執行
  若 cmd1 執行失敗（$? != 0），則 cmd2 不會執行

- cmd1 || cmd2
  此符號則與 && 相反，cmd2 會在 cmd1 執行失敗時才會執行

連續出現：
如果出現這種情形：cmd1 || cmd2 && cmd3
從左而右判斷即可
（若沒有執行到 cmd，則上一個執行的 $? 會直接 pass 到下一個判斷點）
1) 若 cmd1 執行成功，則不會觸發 cmd2，使得 $? = 0 繼續傳遞到 && cmd3
2) 若 cmd1 執行失敗，則觸發 cmd2，後續的 && cmd3 再根據 cmd2 判斷是否執行
